# 论文搭子 ThesisBuddy

面向本科生、研究生和科研工作者的论文全流程工作台。支持从想法立项，也支持导入已有 DOCX 后进行结构拆解、文献审计、数据分析、科研绘图、综合评审和答辩准备。

## 核心工作流

### 从想法开始

1. 创建论文项目，填写研究方向、关键词、学位类型和目标字数。
2. 依次完成选题、开题、文献、结构、写作、打磨、评审和答辩阶段。
3. 各模块产生的大纲、章节草稿、文献证据、数据画像、图表方案和审阅结果会保存到当前项目。

### 导入已有论文

1. 上传 `.docx`；旧版 `.doc` 会提示转换或走服务端兼容转换。
2. 根据 Word 样式校准章、节和小节。
3. 系统建立论文版本、目录树、章节草稿、参考文献、引用位置和项目上下文。
4. 通过 `pipeline/runs` 执行结构拆解与上下文索引，并在上传分解清单中恢复进度。
5. 进入全文审计、知识图谱、科研图表、论文审阅或答辩模块继续处理。

## 主要能力

| 阶段 | 能力 |
| --- | --- |
| 想清楚 | 选题推荐、研究问题、开题大纲 |
| 找资料 | 多源文献检索、参考文献校验、引用位置、全文引用审计、证据矩阵、知识图谱 |
| 搭结构 | DOCX 标题校准、五层正文树、论文大纲、分章草稿、论文版本 |
| 写出来 | 章节扩写、数据分析、科研图表顾问（画像→推荐→代码→质检）、图表计划 |
| 改得好 | 查错、降重、格式检查、术语一致性、段落逻辑、优化建议 |
| 过评审 | 十维论文审阅、论文看板、修改建议 |
| 做答辩 | 英文摘要、答辩 PPT 大纲、答辩材料包 |
| 项目协作 | 论文搭子多源问答、会话记忆、资料库、来源追踪、项目云同步 |

## 费用与余额

- 库内使用厘记账，界面显示为点；`1 点 ≈ 1 元充值额度`。
- 服务端 `CAPABILITY_REGISTRY` 是能力权威来源；打开模块只调用 preflight，真正执行才扣点。
- 计费状态统一为 free / paid / blocked / partial / unavailable。
- 本地确定性步骤、固定价服务器任务和按实际 token 结算的 AI 能力分开计费。
- AI 请求采用预留、执行、按实际用量结算；失败会释放预留额度。
- 余额不足时不会清空输入或丢失已完成的基础结果；有免费降级能力时可先运行基础检查。
- 顶栏余额、账户中心和消费明细可查看余额、免费额度、订单与流水。

## 参考文献与全文审计

- 支持方括号、全角方括号、圆括号、编号列表和合理编号范围。
- DOCX 导入会保留 Word 上标引用语义；普通年份、样本量、百分比和公式指数不会仅因出现数字而被当作引用。
- 引用结果区分已定位、高置信、需复核、未定位和非引用。
- 全文审计按句检查缺引、无效编号、格式不一致和需复核位置，不会自动修改原文。
- 每条问题都可定位原文、查找文献、确认引用、标记无需引用或稍后处理。

## 科研图表顾问

1. 在项目资料库上传 CSV/TSV，或在数据分析页选择本地数据。
2. 先完成数据画像：字段类型、缺失、分布、异常值、分组样本量和相关性。
3. 说明“希望让读者相信什么”，并确认结果变量、分组/时间变量、统计检验、目标章节和期刊规范。
4. 系统给出首选图型、备选图型、变量映射和不推荐方案。
5. 系统会拦截小样本均值柱、饼图、双 Y 轴、3D 图、rainbow/jet 色图、分类折线和未说明误差类型等问题。
6. 保存可复现的 render spec、Python 绘图代码、Figure Artifact、图表计划和质量检查结果。

## 论文搭子助手

论文搭子会在当前项目范围内融合检索：

- 当前论文版本和章节；
- 分章草稿与大纲；
- 已导入文献、论点和证据关系；
- 项目资料库；
- 知识图谱；
- 数据画像、模型结果和图表计划；
- 论文审阅结果。

回答会列出来源类型和结构位置。项目中没有足够证据时，助手会明确说明检查过哪些来源，不会把通用知识伪装成项目证据。

## 首次使用与操作指引

1. 注册并登录后，系统会自动打开首次使用指南；点击“跳过”不会影响项目数据。
2. 右下角 `?` 可随时重新打开指南；顶栏“日志”用于查看版本更新。
3. **从想法开始**：填写研究主题和学科领域，创建项目后按“想清楚 → 找资料 → 搭结构 → 写出来 → 改得好 → 过评审 → 做答辩”推进。
4. **导入已有论文**：上传 DOCX，确认标题层级后再进入引用审计、论文审阅、数据分析或答辩准备。
5. **多数据源分析**：先把 CSV/TSV 上传到当前项目资料库，在数据分析页勾选一份或多份资料；系统先生成各自的数据画像，有公共字段时再执行联合分析。
6. **科研图表顾问**：选择数据后说明希望图表支持的论点，确认变量、统计方法、目标章节和期刊规范，再生成图型建议、绘图代码与规范质检。
7. **点数与计费**：打开模块只检查当前可用性，不会直接扣点；执行收费能力前会显示或确认计费方式。余额、订单和消费记录可在顶栏账户入口查看。

## 快速开始

### 环境要求

- Python 3.11+
- Node.js 22+（运行回归测试）

### 安装和启动

```bash
pip install -r requirements.txt
python start.py
```

浏览器访问 `http://localhost:5000`。

生产环境使用 Gunicorn：

```bash
gunicorn kg_server:app --bind 0.0.0.0:5000 --workers 1 --timeout 180
```

### 必需的生产环境变量

- `JWT_SECRET`
- `ADMIN_SECRET`
- `ADMIN_PASSWORD`
- `DEEPSEEK_API_KEY`
- `DB_PATH`
- `MATERIALS_DIR`
- `SNAPSHOTS_DIR`

## 自动化验证

```bash
node --check app.js
node --check js/app-modules.js
node --check js/modules/project.js
node --check js/modules/literature-workbench.js
python -m py_compile kg_server.py
node tests/run.js
```

GitHub Actions 在推送 `main` 后执行语法检查和回归测试，构建不可变 GHCR 镜像，再更新 Sealos Kubernetes Deployment 并验证 `/health/ready`。

## 数据与部署

生产数据库、资料和论文快照必须位于持久卷：

- `/app/data/thesis.db`
- `/app/data/materials`
- `/app/data/snapshots`

部署采用单 Gunicorn worker 配合 SQLite WAL。扩展为多实例前应迁移到 PostgreSQL，并将限流和任务状态迁移到共享存储。
